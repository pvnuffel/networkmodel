28 /******************************************************************************
29  * These classes have a natural heirarchy of Network > Node > Edge. That means
30  * you probably should not be doing something with nodes unless they already
31  * belong to a network, and you probably shouldn't be handling edges unless
32  * they already belong to at least one node (and a network).
33  *
34  * There are many ways you can constuct a network, but they fall into two
35  * categories: automated (and probably randomized) and manual.
36  *
37  * Start out by getting yourself a network:
38  *
39  * Network my_network("name", false); // "name" is any string you want,
40  * // and false means network is
41  * // undirected (true = directed).
42  *
43  * If you want to automate construction:
44  * my_network.populate(10,000); // Add 10^4 nodes to the network.
45  * my_network.rand_connect_poisson(4); // Connect the nodes using a
46  * // Poisson(4) degree distribution
47  * // and get rid of self-loops and
48  * // multi-edges (done automatically).
49  *
50  */
